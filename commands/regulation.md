# SHIFT AI TIMES 記事レギュレーション

このファイルは記事作成時に必ず遵守すべきレギュレーションです。
構成案作成・本文執筆・入稿の全工程で確認してください。

---

## 構成案

### hタグタイトル

- **区切り文字を使用しない**: パイプ「|」・ハイフン「-」・コロン「:」（全角・半角問わず）
- **h1タイトルは32〜36文字**: 英数字は2文字で1文字換算。「〜を徹底解説」等の補足で多少超えるのは許容
- **h1に対策KWは必須、サブKWは極力含める**: 対策KWはh1タイトルの左側（文の初め）に配置
- **まとめのh2に「まとめ」は入れない**: 「まとめ：」で4文字消費するのは悪手。記事全体を要約していることがわかるタイトルコピーにする
- **よくある質問の見出し**: 「よくある質問」をひとかたまりで使用（間に数字を入れない）

### 見出し構造

- **h2はまとめ以外に3つ以上**
- **同一h2内のh3（同一h3内のh4）は並列関係にする**: 逸脱も不足もない構造にする。「並列関係になるような表現」ではなく「並列関係になるような構造」にする

### 本文内容

- 本文内容フォーマットに則って記載する
- 本文内容をコピペするだけで本文として成り立つレベルで記載する（情報量の過不足なし）

### 内部リンク・CTA

- トピッククラスターの関係を考慮する
  - 絶対入れる記事：ピラーページ＆クラスターページ
  - 文脈にあえば入れる記事：同じピラーページのクラスターページ
  - 入れてはいけない記事：それ以外
- 内部リンク欄には「対策KW/CTA種類 + URL」の両方を記載する（片方のみはNG）
- 挿入箇所を明示する: どのhタグの本文で、どの内部リンク・CTAを挿入するか明記する

## 本文

### 前提

- **紹介する生成AIツールの利用規約を確認する**（商用利用可否、スクリーンショット掲載可否）
- **見直しでは必ず「音読」を行う**（黙読だけでは違和感に気付けない）

### 読者視点での分かりやすさ

- **1文は80文字以下**
- **冗長表現を排除する**:
  - NG: 〜することができる → OK: 〜できる
  - NG: 〜することが可能です → OK: 〜可能です
  - NG: 〜ということです → OK: 〜です
  - NG: 〜があるものです → OK: 〜があります
- **論理飛躍をしない**: 読者の知識レベルを踏まえて論理展開する
- **指示代名詞（こそあど言葉）を多用しない**
- **数値を引用する場合はエビデンスをコメントまたは文中に付ける**
- **固有名詞は正確に記載**: 英語表記のAIツールは2〜3回綴りを確認

### 文章表現

- **差別的・侮蔑的表現は厳禁**
- **本文中で「！」「？」を使用しない**（hタグはOK、「」内はOK）
- **口語表現を使用しない**:
  - NG: こんな / っていう / こんな感じで / あまりにも / いっぺんに / あと / やろうとすると / してやると / してる / 付いて来てない
  - OK: このような / という / このように / 過度に / 一度に / また・さらに / 行おうとすると / すると / している / 付いて来ていない
- **「〜ですよね」「〜ますよ」は使用しない**
- **バイト敬語を使わない**: 「〜になります」→「〜です」
- **表記揺れを起こさない**: 同じ意味の言葉は統一する
- **文章は言い切る**: 「〜と思います」「〜かもしれません」はNG
- **体言止めを使わない**: 文末は「〜です」「〜ます」で終わる
- **「」（）〜：は全角を使う**（「/」はほとんどの場合半角でOK）

### 文法

- **主語述語のねじれがないか確認**: 並列関係や複文節で発生しやすい

### hタグごと

- **h2h3リード文**: 1個目のh3の前に、各h3の内容を簡潔に箇条書きでまとめる
- **まとめ**: 「いかがだったでしょうか？」「いかがでしたか？」で始めない

## 入稿

### 文字装飾

- 色付きアンダーライン：見出しにおいて最重要な箇所
- 太字：次に重要な箇所

### リンク関連

- 全リンクに「新しいタブで開く」を設定
- 内部リンク:
  - テキストリンク：クラスターページ用
  - ブログカード（Simple Blog Card）：ピラーページ用
- 外部リンク: 「>◯◯はこちらから」の形式、「こちら」にリンクを入れる

### CTA

- 本文CTAは1記事に最低1個（リード文・本文1・まとめの計3つは必須）
- 1個の見出しにCTAは1個まで
- 極力CTAと関連記事の内部リンクを同じ見出しに入れない
- CTAの順序: 各CTAに「本文1」「本文2」があり、数字は本文中に設置した順番
- CTA導入文はパターン化されたものを使用する（大城さん作成GPTsで導入文の元を作成し、ユーザーに応じて編集）

### テーブル

- セル内の複数項目はリストスタイルで入れる（「◯◯、△△」は横に伸びるのでNG）
- 「表のセル幅を固定」はOFF
- 「ヘッダーセクション」はON
- モバイルで視認しにくければスクロールをON

### 引用ボックス

- 引用元の記載はボックスの右下に入れる（キャプション機能を使用）

### 画像

- 基本的に画面のスクリーンショットを使用する（フリー画像は使わない）
- Alt属性：画像の内容が明確にわかる内容にする
- 中央寄せにする
- サムネイル：「タイトル」に「記事タイトル【サムネ】」、「代替テキスト」に記事タイトルを入れる

### 改行・空行

- 接続詞があったら改行する（とくに逆接系「しかし」「ただし」）
- モバイルで5行以上になったら改行する（まず文字を削れないかチェック）
- 改行はEnterのみ（Shift+Enterは使わない。※補足・内部リンク・複数出典は例外）
- 空行は入れない

### 表記関連

- 行なう・行う → 行う
- 実は・じつは → 実は
- 分かる → わかる
- 特に → とくに
- 本文・タイトルに不要な半角スペースを入れない（タイトルは【】や記号で区切る）
- プロンプトの記載はコードブロックで統一
- 料金は税込で記載し、その旨を明記する（AIツールは消費税がかからない場合もあるので注意）
- SHIFT AI料金の表記: 必ずパターンを使用し、手入力は行わない
- 木内さんの紹介: 「GMO AI & Web3株式会社の顧問」として記載。GMOロゴや記者会見写真は使用しない（必須と考える場合は大城に相談）
